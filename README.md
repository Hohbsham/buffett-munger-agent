<h1 align="center">🧠 怎么训练一个AI Agent？</h1>
<h3 align="center">—— 以"巴菲特芒格投资智囊"为例</h3>

<p align="center"><em>不需要GPU，不需要标注数据，不需要写代码。<br>一个Markdown文件 + 一个评分习惯 = 一个专家级Agent。</em></p>

---

## 🎯 核心思想

训练AI Agent和训练人一样——**你做的对不对，我告诉你，你下次改进。** 循环足够多次，行为模式就固化了。

```
┌──────────────────────────────────────────────────────┐
│                                                      │
│  Step 1: Agent 分析一个公司/概念                       │
│       ↓                                              │
│  Step 2: 你打分 (1-5⭐) + 一句话指出哪里不对            │
│       ↓                                              │
│  Step 3: Agent 立刻在下一轮应用反馈                    │
│       ↓                                              │
│  Step 4: 满意后，更新 CLAUDE.md 固化行为               │
│       ↓                                              │
│  Step 5: 下次打开文件夹，Agent 自动是训练好的版本        │
│                                                      │
└──────────────────────────────────────────────────────┘
```

**这就是全部。** 没有微调，没有RLHF，没有reward model。你本人就是reward function。你的判断力就是训练信号。

---

## 🏗️ 搭建方法

### 第一步：写一个初始的 CLAUDE.md

告诉Agent它是谁、怎么输出。不用追求完美——第一版一定会被你的反馈改进掉。

```markdown
# CLAUDE.md
你是XXX领域的专家。分析时请包含：1. A 2. B 3. C
```

**就这么多。** 这个文件是Agent的"出厂设置"。关键不是写得多么详细，而是**留出被你批评的空间**。

### 第二步：开始第一轮训练

给Agent一个真实任务，看完输出后**诚实打分**。

```
你："分析一下可口可乐"
Agent：[输出分析]
你：⭐⭐⭐ —— 分析太表面了，没有从生物学角度理解为什么可乐让人上瘾
```

### 第三步：让Agent立刻在下一轮应用

```
你："再来，这次分析茅台，记住上轮的反馈——从跨学科角度挖本质"
Agent：[输出分析——深度明显提升]
你：⭐⭐⭐⭐⭐ —— 社交货币这个角度很好！但还有个问题…
```

### 第四步：把"学到的东西"写进 CLAUDE.md

当某一轮的输出让你满意时，把那个**输出模板**写进CLAUDE.md。下次打开文件夹，Agent自动按这个模板来。

```
轮次 1: 可口可乐 ⭐⭐⭐→ 缺深度，缺跨学科 → 改进CLAUDE.md
轮次 2: 茅台     ⭐⭐⭐→ 缺管理层评估    → 改进CLAUDE.md
轮次 3: 伊利     ⭐⭐⭐⭐→ 缺竞品对比      → 改进CLAUDE.md
轮次 4: 美的vs格力 → 全部到位 → 模板定型
```

## 🔑 为什么这个方法有效？

| 传统AI训练 | 这个方法 |
|------|------|
| 需要几万条标注数据 | 需要你诚实的判断力 |
| 需要GPU集群 | 需要一个Markdown文件 |
| 训练一次几万美元 | 0元 |
| 黑盒，你不知道模型学了什么 | 白盒，CLAUDE.md里写着每一行的来源 |
| 通用，没有你的个人偏好 | **完全个性化**——你讨厌什么，Agent就避免什么 |

**本质区别**：传统训练是让模型模仿"平均人类"，这个方法是让Agent学习**你**。你的品味、你的标准、你的思维框架。

## 🛠️ 进阶：创建可调用的技能

当某个能力稳定后，把它封装成 `/` 命令：

```
.claude/skills/book-tutor/SKILL.md       →  /book-tutor 聪明的投资者 第8章
.claude/skills/deep-analysis/SKILL.md    →  /deep-analysis 茅台
```

技能文件就是一个写了输出模板的Markdown。Agent看到 `/` 命令时加载对应的模板执行。这是**行为封装**——不用每次告诉Agent怎么做，一个命令就够了。

---

## 📂 这个项目的文件

| 文件 | 作用 | 你训练自己的Agent时需要 |
|------|------|------|
| `CLAUDE.md` | Agent行为准则+输出模板 | ✅ **改这个**——换成你的领域和模板 |
| `RL训练系统.md` | 训练日志，记录每轮评分和反馈 | ✅ **复制格式**——记录你自己的训练过程 |
| `.claude/skills/` | 封装好的技能命令 | 🔧 可选——等能力稳定后再封装 |
| `AI_Agent_知识库.md` | 领域知识的提炼 | 🔧 可选——如果领域知识很多，单独存一个文件 |
| `核心投资原则.md` 等 | 具体的领域知识文件 | 🔧 换成你自己领域的知识 |

## 🚀 用这套方法训练你自己的Agent

```bash
# 1. 克隆这个仓库看训练范例
git clone https://github.com/Hohbsham/buffett-munger-agent.git

# 2. 创建你自己的项目
mkdir my-agent && cd my-agent

# 3. 写一个最简单的 CLAUDE.md
echo "# 我的Agent
你是XXX领域的专家。
分析时包含：1. A 2. B 3. C" > CLAUDE.md

# 4. 开始第一轮训练
claude
> "给我分析一下XXX"
> ⭐⭐ —— 这里不够好，因为…

# 5. 重复，直到满意。把最终模板写进 CLAUDE.md。

# 6. (可选) 封装成技能
mkdir -p .claude/skills/my-skill
# 写 SKILL.md
```
